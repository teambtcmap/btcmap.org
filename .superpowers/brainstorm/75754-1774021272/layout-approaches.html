How should sidebar, merchant detail, and map share screen space?

Current problem: Sidebar (320px) + Drawer (400px) = 720px consumed. On a 1366px laptop, over half the map is hidden when viewing merchant details.

A: Drawer Replaces List

Clicking a merchant replaces the list with detail view in the same ~350px panel. Back button returns to list. Google Maps style.

Pros

  • Maximum map space (only 1 panel)
  • Familiar pattern (Google/Apple Maps)
  • Simple navigation model

Cons

  • Lose merchant list context while viewing details
  • Can't compare merchants easily

B: Floating Card Over Map

Sidebar stays visible. Merchant detail floats as a card over the map area. Compact design, can be dragged or repositioned.

Pros

  • See list + detail simultaneously
  • Map still partially visible behind card
  • Flexible positioning

Cons

  • Card covers map markers
  • Requires compact detail design
  • Can feel cluttered

C: Drawer Slides Over List

Merchant detail slides over the sidebar in the same position. List is preserved underneath. Back/close returns to list. Recommended.

Pros

  • Same map space as Approach A
  • List state preserved (scroll position, filters)
  • Natural back navigation
  • Minimal layout changes needed

Cons

  • Can't see list while viewing detail
  • Detail panel narrower than current 400px
